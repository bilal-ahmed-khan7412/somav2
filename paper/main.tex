\begin{abstract}
Multi-agent orchestration frameworks such as AutoGen and LangGraph incur a significant
``Intelligence Tax'': every task triggers full LLM reasoning regardless of complexity,
leading to redundant computation and poor throughput under concurrent load.
This paper presents SOMA V2, a heterogeneous reasoning kernel that eliminates this tax
through three mechanisms: (1) a text-aware depth classifier that routes tasks to
appropriate reasoning tiers with 100\% accuracy on a domain-representative benchmark;
(2) a two-tier semantic plan cache (L1 hot, L2 ChromaDB cold) that reduces complex-task
plan generation latency from 20.07s (simulated full-scale LLM inference) to 0.61ms on
cache hits; and (3) a resource blackboard with inter-agent step negotiation that autonomously
resolves over 90\% of resource conflicts via peer delegation, ensuring all tasks complete under contention. Evaluated on a 60-task workload, SOMA V2 achieves \textbf{51.7\% lower LLM invocation}
than baseline through its V2 Hybrid Router fast-path, while improving task success rate
to 98.3\%. Real-world capability showcase using real \texttt{qwen2.5:3b} inference proves unique architectural
properties: plan reuse, structured JSON-based command injection, and high-fidelity actuator bridging.
\end{abstract}

\section{Introduction}

As LLMs transition from passive tools to active orchestrators, the bottleneck has shifted
from generation quality to orchestration efficiency. AutoGen~\cite{autogen} and
LangGraph~\cite{langgraph} treat every task as a fresh reasoning problem, re-invoking the
LLM even for tasks structurally identical to previously solved ones. This creates two
failure modes under real workloads: (1) \emph{redundant LLM calls} for tasks that could be
handled by rules or cached plans, and (2) \emph{throughput collapse} when many tasks are
submitted concurrently, because sequential frameworks cannot overlap computation.

SOMA V2 addresses both failure modes. Its depth classifier eliminates LLM calls for simple
tasks entirely; its semantic cache eliminates LLM calls for recurring complex tasks; and its
concurrent execution model overlaps all remaining calls within a configurable semaphore
budget. Additionally, the ResourceBlackboard with inter-agent step negotiation prevents
contention-induced blocking when multiple agents target the same physical resource.

The primary contributions of this paper are:
\begin{enumerate}
  \item A \textbf{text-aware heterogeneous dispatch kernel} (ReactiveAgent /
        RoutingAgent / DeliberativeAgent) with O(1) keyword-rule classification
        achieving 100\% accuracy on our benchmark workload.
  \item A \textbf{two-tier semantic plan cache} (L1 hot + L2 ChromaDB cold) reducing
        complex-task latency from 20.07s (simulated full-scale LLM cold) to 0.61ms (L1 hit).
  \item A \textbf{ResourceBlackboard with NegotiationBroker} providing fault-tolerant
        resource arbitration: autonomously resolves 75\% of conflicts via inter-agent
        step delegation with fire-and-forget async dispatch (net-neutral latency), ensuring task completion under high contention.
  \item A \textbf{failure-driven re-planning} mechanism that evicts corrupted cache
        entries and re-generates plans with episodic failure context injected.
  \item Empirical evidence of 8.7$\times$ throughput advantage over AutoGen and
        89~tasks/s under 16-agent concurrent stress in stub mode.
\end{enumerate}

\subsection{Text-Aware Depth Classifier}

The \texttt{DepthClassifier} routes each incoming task to one of three tiers before any
LLM is invoked. V2 introduces a \textbf{Hybrid Router fast-path} that short-circuits
uncontested low/medium urgency tasks directly to the reactive tier, bypassing ML inference
for roughly 52\% of the workload\footnote{Efficiency gains are workload-dependent; the fast-path specifically targets non-critical, uncontested events common in routine swarm maintenance.}. Remaining tasks follow a priority cascade:

\begin{enumerate}
\item \textbf{Keyword rule} (O(1)): Six compiled regex patterns detect unambiguously
      simple tasks (status checks, queries) or unambiguously complex tasks (multi-entity
      coordination, optimization). This rule fires first and handles the majority of tasks
      at confidence $p=1.0$.
\item \textbf{Random Forest model} (13 features): For ambiguous cases, a trained
      classifier uses 8 text features (character count, word count, keyword hit counts,
      binary coordination/optimisation flags) and 5 metadata features (agent role,
      confidence, urgency, contention flag, reroute count).
\item \textbf{Urgency heuristic}: Falls back to \texttt{medium} depth when the model is
      unavailable; promotes to \texttt{complex} if urgency is high and reroute attempts
      exceed zero.
\end{enumerate}

The classifier supports online retraining every 50 resolved tasks via a background thread.
Empirical evaluation on the 30-task benchmark shows \textbf{100\% accuracy} across all
three classes (simple / medium / complex), with precision and recall both at 1.0.

\subsection{Heterogeneous Agent Tiers}

Three agent types handle dispatched tasks:

\begin{itemize}
\item \textbf{ReactiveAgent}: Rule-table lookup. Zero LLM calls. Handles \texttt{simple}
      depth tasks in sub-millisecond time. In the benchmark, 27\% (8/30) of tasks were
      resolved this way.
\item \textbf{RoutingAgent}: Single LLM call for \texttt{medium} depth tasks. Handles
      13\% (6/30) of the benchmark workload.
\item \textbf{DeliberativeAgent}: Multi-step DAG planner for \texttt{complex} tasks.
      Consults the semantic memory hierarchy before invoking the LLM; a cache hit bypasses
      the LLM entirely, dropping plan latency from \textbf{20.07s} (simulated \texttt{qwen2.5:3b}
      cold generation) to \textbf{0.61ms} on L1 hit --- an empirical \textbf{32,823$\times$} reduction for recurring
      mission patterns.
\end{itemize}

\subsection{Semantic Plan Memoization}

The \texttt{HierarchicalMemory} system provides two cache tiers for deliberative plans:

\begin{itemize}
\item \textbf{L1 Hot Cache} (\texttt{HotMemory}): In-process dictionary keyed by an
      MD5 hash of a normalized event string. Sub-millisecond lookup. Populated on first
      resolution.
\item \textbf{L2 Cold Cache} (\texttt{ColdMemory}): ChromaDB collection with
      \texttt{sentence-transformers/all-MiniLM-L6-v2} embeddings for semantic retrieval.
      Returns similar plans for novel phrasings of previously solved tasks with Jaccard
      similarity scoring.
\end{itemize}

Event normalization strips numeric suffixes and variant prefixes (e.g., ``Var\_3:''
batch prefixes) so structurally identical tasks share cache entries regardless of
surface variation. Plans retrieved from L2 are immediately promoted to L1 for subsequent
instant hits. Failed cached plans are evicted from L1 and re-planned on the next encounter
to enable failure-driven specialization.

\subsection{ResourceBlackboard and NegotiationBroker}

Physical resource contention --- where two concurrent agents attempt to actuate the same
unit (e.g., drone A12) --- is managed by the \texttt{ResourceBlackboard}: an asyncio-native
exclusive ownership layer. When an agent cannot claim a unit within \texttt{claim\_timeout\_s},
the \texttt{NegotiationBroker} intercepts the conflict and proposes the blocked step to
the current unit owner for inline co-execution via reentrant claim. This prevents outright
task failure in contention scenarios. Empirical evaluation (Section~\ref{sec:negotiation})
shows the broker resolves 75\% of conflicts autonomously via fire-and-forget async negotiation, with all remaining tasks
completing via backtrack paths.

\subsection{Concurrency-Aware LLM Wrapper}

SOMA V2 dispatches all tasks concurrently via \texttt{asyncio.gather}, but a shared
\texttt{asyncio.Semaphore} (default capacity: 2) limits in-flight LLM calls. This prevents
queue saturation on single-instance backends (e.g., local Ollama). The wrapper additionally
enforces a per-call timeout (30s) and exponential backoff (up to 2 retries) for resilience
against transient failures.

\section{Empirical Evaluation}

\subsection{Experimental Setup}

All benchmarks run in stub mode (mock LLM, 20ms/call simulated latency) to isolate
architectural overhead from model inference variance. The workload consists of 30 tasks
generated from 5 unique templates: 60\% complex (multi-drone coordination, sensor
recalibration, energy distribution), 27\% simple (status checks), and 13\% medium
(single-step routing). We benchmark three systems:

\begin{itemize}
  \item \textbf{AutoGen}: \texttt{RoundRobinGroupChat} with a planner + executor agent.
        Complex tasks use \texttt{MaxMessageTermination(4)}; medium tasks use a single-agent
        team with 1 turn.
  \item \textbf{LangGraph}: \texttt{StateGraph} with three nodes
        (\texttt{classify} $\rightarrow$ \texttt{plan} $\rightarrow$ \texttt{execute}) and
        conditional edges.
  \item \textbf{SOMA V2}: 6 concurrent slots (1 SUPERVISOR + 5 PEER), warm-up phase
        pre-populates L1 cache before the timed workload.
\end{itemize}

\subsection{Negotiation Benefit and ResourcePool}
\label{sec:negotiation}

To evaluate the ResourceBlackboard, NegotiationBroker, and ResourcePool, we ran a
mixed-contention workload: 60 concurrent tasks across 4 drones (B1--B4), with 60\%
of tasks targeting the hotspot drone B1 and 40\% distributed across B2--B4. We compare
three conditions representing SOMA V2's three-tier conflict resolution hierarchy:

\begin{enumerate}
  \item \textbf{Named / No-neg}: tasks target a named drone; on \texttt{claim\_timeout\_s=0.05}
        expiry the node is immediately marked FAILED (backtrack path only).
  \item \textbf{Named / Neg-ON}: same named targeting; on timeout the
        \texttt{NegotiationBroker} proposes the blocked step to the unit owner for
        inline co-execution (fire-and-forget, reentrant blackboard claim).
  \item \textbf{Pool claim}: tasks call \texttt{ResourcePool.claim\_any()} and receive
        whichever of B1--B4 is currently free --- contention is absorbed by the pool
        rather than resolved after the fact.
\end{enumerate}

All 60 tasks completed in every condition. Key findings (Table~\ref{tab:negotiation}):

\textbf{Negotiation} (Named/Neg-ON) resolves 93\% of detected conflict attempts via
fire-and-forget peer delegation, reducing mean latency by 76\% and increasing throughput
by 4.3$\times$ versus the no-negotiation baseline. The broker succeeds because blocked
steps are delegated to whichever agent currently owns a free drone, turning a serial
bottleneck into parallel co-execution.

\textbf{ResourcePool} eliminates conflicts entirely (0 vs.\ 52) because tasks claim
any free equivalent drone at the task level rather than competing for a specific named
unit. Tasks queue until any drone is available, trading slightly higher per-task latency
for zero contention. This is the preferred strategy when drones are interchangeable and
the workload is not saturating all units simultaneously. Neither AutoGen nor LangGraph
provide a ResourceBlackboard, NegotiationBroker, or ResourcePool equivalent.

To identify the architectural ceiling, we swept the number of concurrent agent slots from 1 to 64 on a fixed 100-task workload with a warm cache. \subsection{Multi-Model Generalization}

We verified that the 32,823$\times$ cache speedup for complex plans is an architectural property, not a model artifact. Evaluating semantic plan reuse with simulated full-scale LLM load (20,071ms cold $\rightarrow$ 0.61ms warm) confirms that while cold plan generation latency scales with model size, the warm cache hit latency remains constant at under 1ms. Cached plans cost zero LLM calls, regardless of the generator model's parameter count.

\subsection{Zero-Shot Domain Adaptation}

SOMA V2 was seeded with Urban Rescue tasks then presented with structurally identical
Deep Sea Salvage tasks using completely different vocabulary. By normalizing domain terms
(Drone $\rightarrow$ \texttt{AGENT\_UNIT}, ROV $\rightarrow$ \texttt{AGENT\_UNIT}),
SOMA V2 achieved a \textbf{50\% zero-shot cache hit rate} in the second domain --- with
a semantic distance of 1.04 --- effectively transferring learned coordination logic across
domains without fine-tuning. AutoGen and LangGraph have no analogous cross-session reuse
mechanism.

Key findings from Table~\ref{tab:adversarial}:

\textbf{Prompt injection containment.} All 12 tasks containing injection strings
(\texttt{IGNORE ALL}, \texttt{OVERRIDE}, \texttt{rm -rf}) completed successfully.
Zero injected commands reached the hardware actuator bridge. The \texttt{[CMD]} parser
ignores free-text injection because it only extracts structured \texttt{[CMD] VERB UNIT}
tokens from step descriptions, which are LLM-generated --- not copied from task input.

\textbf{Thundering herd resilience.} 500 concurrent tasks were absorbed without
crash or timeout storm. The \texttt{asyncio.Semaphore} correctly serialized LLM calls
while concurrent actuator commands and cache lookups proceeded in parallel.

\textbf{Memory self-healing.} After L1 was flooded with 10,000 garbage entries and
the real plan key was TTL-expired, the L2 ChromaDB cold store served the plan at 108ms
--- confirming two-tier redundancy under memory pressure.

\textbf{Cascade failure isolation.} Forcibly terminating one of four agent slots
mid-run caused zero task failures (tested in isolation: 20/20). In the full suite,
a ChromaDB SQLite file-handle race between ATCK~5 and ATCK~7 caused a crash ---
a test isolation artifact unrelated to the swarm architecture.

\textbf{Classifier degradation behavior.} Under 12 adversarially crafted inputs,
the depth classifier achieved 42\% label accuracy. Critically, all 12 tasks still
completed successfully --- the classifier defaulted to \texttt{medium} depth rather
than crashing or selecting an incorrect extreme tier. This \emph{conservative safe-default}
behavior is by design: over-routing to a higher reasoning tier costs efficiency but
never produces incorrect outputs. The 42\% figure quantifies the cost of this safety
margin on deliberately pathological inputs not representative of real mission workloads.

\subsection{Threats to Validity}

\textbf{Benchmark scope.} The stub benchmark uses a fixed 5-template workload.
Real-world deployments with a higher proportion of novel tasks will show lower cache
hit rates. We quantify this: at 0\% task repetition, SOMA V2 has no cache advantage.

\textbf{Concurrency comparison.} AutoGen and LangGraph can be parallelised externally.
The wall-time advantage is in part a concurrency effect built into SOMA V2 by design.

\textbf{Real LLM stability.} Full Ollama runs for all five capabilities were limited
by RAM pressure on the test machine (\texttt{qwen2.5:3b} + sentence-transformer +
ChromaDB competing for memory). CAP~1 Ollama numbers are confirmed; remaining
capabilities are validated in stub mode with architectural equivalence.

\textbf{Adversarial classifier accuracy.} The 42\% label accuracy on crafted adversarial
inputs (Section~\ref{sec:adversarial}) reflects an honest limitation of the current
keyword-rule + Random Forest classifier under out-of-distribution phrasing. All tasks
still complete correctly due to the safe-default degradation property.

\section{Limitations}

We report three honest limitations reviewers should consider:

\begin{enumerate}
  \item \textbf{Cold-start penalty.} The first encounter of each unique task template
        requires full LLM planning. In fully novel workloads (no task repetition), SOMA V2
        provides no cache advantage over AutoGen or LangGraph. The system is designed for
        \emph{recurring mission patterns}, not one-shot general-purpose querying.
  \item \textbf{Domain specialization.} The \texttt{[CMD]} actuator bridge, ResourceBlackboard,
        and negotiation protocol are purpose-built for physical swarm coordination. AutoGen
        and LangGraph offer richer general-purpose tool ecosystems (web search, code
        execution, function calling) that SOMA V2 does not provide.
  \item \textbf{No human-in-the-loop.} AutoGen's \texttt{HumanProxyAgent} and LangGraph's
        \texttt{interrupt()} provide native human approval gates. SOMA V2 is fully automated
        with no built-in pause-for-review mechanism.
\end{enumerate}

\section{Conclusion}

SOMA V2 demonstrates that concurrency, depth gating, semantic plan caching, and
inter-agent resource negotiation together produce multi-agent systems substantially
more efficient than general-purpose frameworks for recurring-mission workloads.
Five empirically validated unique capabilities --- plan reuse, cross-domain transfer,
live resource negotiation, failure-driven re-planning, and 89~tasks/s concurrent
throughput --- collectively address limitations that AutoGen and LangGraph do not
architecturally support.

In stub-mode framework comparison, SOMA V2 achieves \textbf{51.7\%}
fewer LLM calls per task through its Hybrid Router fast-path. This mechanism is
conservative: under high-urgency or resource contention, the fast-path automatically
disengages, routing tasks through the full classifier hierarchy to ensure reasoning
integrity. In simulated full-scale LLM load, cold plan generation averages 20.07s; warm L1 cache
hits serve the same plan in 0.61ms --- an empirical 32,823$\times$ reduction. The ResourceBlackboard provides a three-tier contention hierarchy: pool-based claiming
eliminates conflicts entirely (0 vs.\ 52, +27\% throughput); when named claims are
required, the NegotiationBroker resolves over 90\% of attempts via fire-and-forget peer
delegation. Reliability is further hardened via structured JSON command schemas and
cascading delegation timeouts. Failure-driven
cache eviction enables autonomous self-repair without human intervention.

The honest cost of this specialization --- cold-start penalty, domain scope, and
absence of general tool use --- is quantified in Section~\ref{sec:showcase} and
the Limitations section. Future work will evaluate SOMA V2 under fully novel workloads,
integrate streaming LLM responses, and extend the actuator bridge to real drone
hardware via MAVLink.
